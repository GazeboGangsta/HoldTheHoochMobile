### M5a — Programmatic particles (closed)

Design spec at [docs/superpowers/specs/2026-04-20-m5a-particles-design.md](superpowers/specs/2026-04-20-m5a-particles-design.md), plan at [docs/superpowers/plans/2026-04-20-m5a-particles.md](superpowers/plans/2026-04-20-m5a-particles.md). Nine tasks via subagent-driven development + three post-review / tune commits:

- **`SparkleBurst` factory** at [lib/components/sparkle_burst.dart](../lib/components/sparkle_burst.dart) — static `emit(kind, at)` returns a self-removing `ParticleSystemComponent`. Per-kind config (herb green 6×4, hops gold 10×6, potion blue 16×8 + expanding halo). Closed-form exponential-decay motion so visual feel is dt-stable across framerates.
- **`SplashEmitter` component** at [lib/components/splash_emitter.dart](../lib/components/splash_emitter.dart) — hybrid burst-on-threshold-crossing + rate-scaled continuous trickle + dramatic game-over flourish. Edge-detection + accumulator-based rate math. Owns its PSCs as children (Flame 1.37 test-harness concurrent-mod workaround); `reset()` clears own children + pending queue for restart integration.
- **`Gurgles.tankardRimWorldPosition`** public method + `@visibleForTesting Gurgles.rimOffsetFromTankardCenter` static pure fn. Fixed post-playtest: uses `t.absoluteCenter` (Flame handles bottomCenter-anchor math correctly) and `t.size.y / 2` for the rim-rotation radius (tankard sprite is wider than tall).
- **`CollectibleManager.onPickup`** signature widened from `(int, Vector2)` → `(int, Vector2, CollectibleKind)`; `GameScene.onLoad` emits `SparkleBurst` alongside the existing `ScorePopup` on pickup.
- **`GameScene._end(String, {bool fromSpill = false})`** — on spill death, fires `splashEmitter.emitGameOverBurst()` then a 600 ms `Future.delayed` overlay. Uses `isAttached` (not `isMounted`) as the guard so the callback doesn't race a Navigator-pop. Obstacle-hit path unchanged (instant overlay).
- **Restart cleanup** in `GameScene.restart()`: scene-level `children.whereType<PSC>()` cleanup (for sparkle PSCs) + `splashEmitter.reset()` (for emitter-owned splash PSCs). Integration test uses `descendants()` to verify both paths.

Device playtest surfaced one real bug (splash emitting from wrong world position) and one real tune (sparkle radii doubled). User deferred game-over-burst tuning to Post-V1 "whole-screen death effect" idea — current burst reads as subtle on-device.

### First TestFlight build

First iOS IPA built, signed, and uploaded to App Store Connect on 2026-04-21 via CodeMagic (`ios-release` workflow on tag `v0.1.0`, build id `69e6862caaf0f6fde43c5bdb`). The road to getting there ate ~2 hours:

- CodeMagic's documented "automatic iOS code signing" does not work for first-time Apple Developer accounts. Multiple failure modes, well documented now in [STATUS.md Gotchas](STATUS.md#gotchas-dont-repeat-these) and [SIGNING.md](SIGNING.md).
- The working path: generate RSA private key + CSR on Windows via OpenSSL → upload CSR to Apple Developer → download `.cer` → bundle cert+key into `.p12` → upload `.p12` to CodeMagic Code signing identities → manually create App Store provisioning profile at Apple Developer → upload `.mobileprovision` to CodeMagic's profile store. Then the canonical `ios_signing` declarative block in the yaml finds both and signs cleanly.
- Local signing artefacts at [ios-signing/](../ios-signing/) (gitignored) include the private key, CSR, `.cer`, `.p12`, and password. Keep these backed up — losing the private key means re-doing Apple cert registration.

### If the TestFlight build fails processing

Apple sometimes rejects first builds for things like missing app icon (we're still on default Flutter icons), missing PrivacyInfo.xcprivacy, or export-compliance miscommunication. Apple emails a rejection reason. Most fixes are small and can be patched in a new `v0.1.1` tag.
