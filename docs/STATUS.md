# Project Status — HoldTheHooch Mobile

_Last updated: 2026-04-14_

Snapshot of where the project is, what's working, what isn't, and what to pick up next session.

## TL;DR

Working Flutter + Flame 2D side-scrolling endless runner on Android. Installable debug APK running on Pixel 8 Pro via wireless debugging. Core loop is playable: tap right to jump, drag left to balance the hooch, don't hit obstacles, don't spill. Real SVG art from the web game is in, parallax backdrop scrolls, score submits to `gurgles.beer`. **Not yet tested on iOS.**

## What's playable today

- **Menu** → name entry (persisted locally) → **Start** → game scene → **Game Over** overlay → **Retry** / **Menu**.
- Gurgles runs (placeholder: sprite swaps to jump pose mid-air — no animated run cycle yet).
- **Tap right half** → variable-height jump (hold for higher).
- **Drag left half horizontally** → counter-tilt the hooch toward center.
- Passive hooch wobble builds tilt over time; every jump adds a random tilt impulse.
- **Spill meter** (red bar below score) fills when `|tilt| > 0.7` and drains when `|tilt| < 0.4`. 100% = game over.
- Obstacles (root / rock / mushroom / log) spawn at speed-dependent intervals and end the run on collision.
- Parallax: distant mountains (0.15x) + forest trees (0.45x) + tiled ground (1.0x).
- Score: +10 per second elapsed, shown top-center.
- Difficulty curve: scroll speed ramps 1.0x → 2.0x over 180 seconds.
- **Game Over** overlay shows score, persists best-score locally, submits to `https://gurgles.beer/api/scores` (silently; offline still shows score).

## Gotchas learned this session (don't repeat these)

- **Flame 1.37's default CameraComponent viewfinder is center-anchored.** World-(0,0) lands at screen center. Pin `camera.viewfinder.anchor = Anchor.topLeft` in `onLoad` if you want top-left origin semantics.
- **`Theme.of(context)` cannot be called from `initState`.** Use `defaultTargetPlatform` from `package:flutter/foundation.dart` if you only need the platform name.
- **Scaffold + keyboard = shrunken GameWidget.** If the name-entry TextField is still focused when navigating to the game, the Scaffold body resizes to avoid the keyboard inset and Flame's canvas ends up ~2/3 height. Fix: `FocusManager.instance.primaryFocus?.unfocus()` before pushing, and `resizeToAvoidBottomInset: false` on the game's Scaffold.
- **Strict `y >= groundY` checks break on subpixel drift.** Add a couple-pixel tolerance or clamp on snap.
- **Prefer `Listener` + `onPointerDown` over `GestureDetector` or Flame's `TapCallbacks`** when embedding Flame in Flutter — pointer events are raw and bypass gesture arenas, so taps always reach you.
- **`debugPrint`/`dev.log` from a release APK on Android 15 Pixel sometimes doesn't surface through `adb logcat` reliably.** On-screen diagnostic HUD (live `TextComponent`) is faster than chasing logs.
- **`flame_svg`'s `Svg.render(canvas, size)` doesn't take a position argument.** Use `canvas.translate()` for layered tiling.
- **`flame` / `flame_svg` ship without raster-image parallax helpers that accept SVG.** Hand-rolled `ParallaxLayer` wraps `Svg.render` in a manual tile loop.
